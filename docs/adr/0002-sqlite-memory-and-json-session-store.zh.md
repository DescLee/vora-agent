# ADR 0002：长期记忆使用 SQLite，会话状态使用 JSON 文件

## 状态

已采纳。

## 背景

当前项目是本地单用户 TUI Agent Runtime。它需要保留用户偏好、项目摘要、产物摘要和明确决策，也需要支持会话 list/resume。但第一版不需要多租户、高并发、复杂权限或语义向量检索。

## 决策

- 长期记忆使用标准库 `sqlite3`。
- 会话状态使用 JSON 文件持久化。
- 记忆写入前做敏感信息过滤，避免 token、password、secret 等内容进入长期存储。
- 记忆检索采用关键词与标签匹配，保持实现可解释。

## 取舍

优势：

- 不引入额外服务，面试演示和测试环境稳定。
- 数据格式简单，可直接检查和调试。
- SQLite 便于后续增加索引、时间排序和迁移。

代价：

- 关键词检索无法覆盖语义相似问题。
- JSON session 不适合多用户并发写入。
- 缺少集中式审计、权限和生命周期管理。

## 后续

生产化可以演进为：

- 会话状态迁移到服务端数据库。
- 长期记忆增加 embedding 检索、过期策略和分层记忆。
- 敏感数据过滤升级为统一 redaction pipeline，并覆盖日志、memory、artifact。
